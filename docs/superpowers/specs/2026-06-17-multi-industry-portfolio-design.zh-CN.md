# 多行业作品集站点设计说明

## 目标

把当前只有牙医案例的单页站点，扩展成一个放在同一套 Next.js 项目和同一域名下的客户展示型作品集。站点需要包含：

- 一个作品集首页
- 三个完整行业案例页

三个行业案例分别为：

- 牙医诊所
- 律师事务所
- 房产经纪

最终效果应更像一个精品网站工作室的展示站，而不是模板市场。

## 已确认方向

本次确认的是“混合作品集”方向：

- 根路径首页承担你的工作室门面角色
- 每个行业案例页都要像一个真实、可信、可给客户直接看的成品官网
- 每个案例页在主体内容体验结束后，轻量补充一段“这个案例体现了什么能力”的总结

首页整体视觉气质采用现代创意工作室风格：更强调排版、节奏、留白、案例呈现与设计感。三个案例页则分别切换到各自行业更匹配的视觉语言，同时保持属于同一位设计师/同一工作室产出的统一完成度。

## 暂定作品集品牌

首页使用一个默认的作品集品牌作为占位：

- 品牌名：`Linea Studio`
- 品牌定位：为服务型行业打造高品质、重转化、可上线的品牌官网

这个名字只是当前设计与文案默认值，后续可以单独改名，不影响整个信息架构。

## 信息架构

站点采用四个顶层路由：

- `/` - 作品集首页
- `/dental` - 牙医诊所案例页
- `/law` - 律师事务所案例页
- `/real-estate` - 房产经纪案例页

当前已经完成的牙医站不再继续作为根路径首页，而是迁移为 `/dental` 案例页。

## 首页结构

首页需要同时承担“介绍你自己/你的工作室”和“引导客户进入案例”的双重角色，不能做成简历页，也不能做成普通 agency 套壳页。

### 模块顺序

1. 页头
2. 首屏 Hero
3. 精选案例区
4. 能力说明区
5. 合作流程区
6. 联系/咨询区
7. 页脚

### 页头

- `Linea Studio` 品牌字标
- 指向首页各区块的主导航
- 一个清晰的 CTA，例如 `查看案例` 或 `发起项目`

### 首屏 Hero

作用：

- 快速建立审美、专业度和可信度
- 说明这是一个覆盖多个服务型行业的网站作品集
- 引导客户去看案例

内容方向：

- 一句简短但有力度的价值主张
- 一段说明性文案，强调定制官网设计、响应式实现、SEO 基础结构与上线交付能力
- 两个按钮：一个指向案例区，一个指向联系/咨询

### 精选案例区

这是首页最核心的成交区。

三张案例卡片都应包含：

- 行业标签
- 案例品牌名
- 一句简短定位文案
- 大尺寸预览视觉
- 进入完整案例页的按钮/链接

整体呈现应偏大图、偏编辑感、偏作品展示，不要做成普通博客缩略卡。

### 能力说明区

总结这个作品集所体现的能力：

- 品牌首页设计
- 响应式前端实现
- SEO 友好的页面结构
- 面向转化的咨询/联系设计
- 可部署、可交付的网站成品

这一部分应该简洁、可信，不做成廉价图标九宫格。

### 合作流程区

用一个轻量但专业的小工作室流程，强化客户对你的信任：

1. 沟通需求
2. 确定方向
3. 设计与开发
4. 上线交付

这部分要简练，不做成长篇服务介绍。

### 联系区

提供一个真实但轻量的咨询入口：

- 一段邀请咨询的文案
- 联系方式摘要
- 一个简单表单或 CTA 区块

## 案例页策略

每个案例页必须先像一个真实行业官网，之后才轻量透露它是你的作品案例。

因此，每个案例页都应具备：

1. 一个完整可信的行业首屏
2. 符合行业逻辑的内容区块
3. 一个联系/咨询区
4. 靠近页面底部的一小段案例总结，说明这个案例体现了哪些设计与商业能力

## 三个行业案例定义

### 牙医案例：`晴禾齿科`

路由：

- `/dental`

状态：

- 复用当前已完成牙医站作为基础

气质关键词：

- 明亮
- 安心
- 洁净
- 家庭友好
- 专业但温和

主要内容：

- 首屏
- 服务介绍
- 信任/流程
- 联系与预约

页面底部的案例总结应强调：

- 医疗行业的信任感建立
- 清晰且利于转化的信息层级
- 温和但专业的品牌表达
- 强移动端表现

### 律师事务所案例：`衡正律师事务所`

路由：

- `/law`

气质关键词：

- 权威
- 冷静
- 高信任
- 高端商务
- 极简严肃

建议模块：

- 带咨询 CTA 的强势首屏
- 业务领域
- 事务所优势/定位
- 律师团队或核心成员
- 联系/咨询入口

视觉系统方向：

- 深墨色、石板灰、暖白色、克制金属感点缀
- 强排版
- 少装饰
- 稳重且有力量的留白与内容框架

页面底部案例总结应强调：

- 高信任服务行业的定位表达
- 高端专业官网呈现
- 法律服务内容层级组织
- 高客单价咨询线索承接

### 房产经纪案例：`珑域地产`

路由：

- `/real-estate`

气质关键词：

- 城市感
- 品质感
- 高端住宅导向
- 专业顾问感
- 视觉展示力更强

建议模块：

- 以优质房源/城市生活方式为切入的首屏
- 精选房源或展示房源
- 买房与卖房服务
- 经纪人/顾问信任区
- 看房/咨询入口

视觉系统方向：

- 精致中性色底
- 深炭黑、偏墨色与低饱和铜色或深蓝点缀
- 更强调高质量大图
- 兼顾浏览效率与高级感的房源卡片/展示结构

页面底部案例总结应强调：

- 房产行业的视觉展示力
- 有利于咨询转化的房源信息布局
- 本地品牌感与品质感塑造
- 移动端浏览与咨询路径优化

## 共享设计系统规则

整个站需要在作品集层面保持统一完成度，但不能把三个行业硬套成一个模板。

### 共享规则

- 一致的排版纪律
- 一致的留白逻辑
- 一致的按钮质感与交互完成度
- 一致的圆角克制策略
- 一致的页脚与跨案例导航方式
- 一致的 SEO 与 metadata 架构

### 允许按行业变化的部分

- 配色
- 首屏视觉表达
- 模块排序细节
- 图片占比与密度
- 文案语气

## 导航模型

作品集需要让用户既能沉浸看案例，又不会迷路。

### 首页导航

首页导航指向：

- 关于/能力
- 案例
- 流程
- 联系

### 案例页导航

每个案例页都应提供：

- 适合该案例网站自身的导航结构
- 一个较轻的返回作品集首页入口
- 靠近页底的其他案例跳转区

这样既不破坏案例沉浸感，也方便作品集浏览。

## SEO 要求

每个路由都应有自己独立的 metadata。

### 首页 SEO

首页关键词方向应围绕：

- 高品质网站作品集
- 服务型企业官网设计
- 响应式商业网站案例

### 案例页 SEO

每个案例页都应有：

- 独立标题
- 独立描述
- canonical 路由
- Open Graph 信息
- 语义化标题结构
- 有意义的图片 alt 文本

## 图片与视觉资产策略

和当前只有牙医案例的状态相比，这个项目需要更完整的作品集级视觉资产系统。

### 首页资产

- 一张偏工作室气质的首屏主视觉，或更抽象的编辑感构图
- 三张案例预览主视觉

### 案例页资产

- 保留现有牙医站图片资产
- 为律师与房产案例补充匹配的高质量视觉
- 维持真实、高级、现代的整体感觉
- 尽量避免廉价 stock 图拼贴感

## 组件架构

实现层应从当前单页结构，升级为一个小型可复用系统。

预期可复用部分包括：

- 作品集页头
- 页脚
- Hero 容器
- 案例卡片
- 能力说明区
- 流程区
- 联系/咨询区
- 案例总结区

行业差异内容应放在类型明确的数据结构或各自路由组件中，而不是继续塞进一个超大的 `page.tsx` 文件。

## 数据流与行为

本阶段仍然是静态内容站点。

- 不需要 CMS
- 不需要登录态
- 联系表单可以保持 UI 级别，暂不接真实提交服务
- 导航与路由跳转应简单、明确、可靠

## 错误处理

即使未来个别图片或内容变更，构建和页面也应保持稳定。

- 图片路径必须明确且经过校验
- 内部路由链接在改造后必须全部检查
- 即使联系表单仍为静态 UI，各页面也应完整可用

## 响应式要求

整站必须在手机、平板、桌面端都保持清楚和专业。

关键目标：

- 首页首屏在手机上依旧高级、清晰
- 案例卡片在小屏幕下不变得拥挤
- 律所与房产页在移动端仍保持高端气质
- 案例页 CTA 与联系区在手机上依然好用

## 测试与验收

实现后既要做代码层验证，也要做浏览器体验验证。

### 代码验证

- `npm run lint`
- `npm run build`

### 浏览器验证

- 首页桌面端与移动端
- 三个案例页桌面端与移动端
- 首页与案例页之间的导航
- 联系区可见性与布局
- 跨案例跳转区

### 设计验证

重点确认：

- 首页是否明确像一个创意工作室作品集
- 每个案例页是否足够像真实客户官网
- 三个行业之间的气质差异是否明显
- 三个案例整体是否依然像同一个有能力的人/团队做出来的

## 本阶段不包含

本轮先不做：

- CMS 接入
- 真实预约后端
- 真实房源数据库
- 法律咨询自动化流程
- 多语言
- 数据分析后台

## 成功标准

当满足以下条件时，视为本轮成功：

- `ai1234567.xyz` 从单一行业站升级成完整作品集
- 首页能够清楚传达你的能力和审美
- 三个行业案例都足够成熟，可以直接拿给潜在客户看
- 整站响应式良好、可部署、并具备基础 SEO 完整度
